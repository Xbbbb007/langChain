# Code Comments Quality Check Skill (comments-check)

此技能用于对指定的代码文件进行多维度的注释检查与质量审计，确保代码的高可读性与通俗性。

## 何时使用

1. 用户输入 `/comments-check <file_path>` 或请求“检查注释”、“审查注释质量”。
2. 在进行代码库大版本交付或重构验收，需要评估代码注释率与易读性时。

## 审计与检查标准

1. **密度检查（完整性）**：
   * 核心公式/规则：每 9 行代码正文应至少匹配 1 行注释（注释行数占总有效代码行的 11.1% 以上）。
   * 检查所有函数、类是否包含定义级的文档字符串（Docstring）或前置注释。
2. **一致性检查（准确度）**：
   * 细读现有注释，比对底层代码实现，确认是否存在“注释写着 A 逻辑，代码实际上写了 B 逻辑”的不匹配现象。
3. **小白视角（通俗度）**：
   * 审查注释是否能将复杂的业务术语、API 结构或逻辑分支用简单直白的语言表述清楚（注重解释“为什么这样做”，而非单纯复述“代码做了什么”）。

## 任务工作流

1. **执行静态分析**：
   运行静态检查脚本，统计代码行、注释行以及缺少文档说明的函数/类：
   ```bash
   .venv\Scripts\python .agents/plugins/testing-plugin/skills/comments-check/scripts/check_comments.py <target_file_path>
   ```
2. **进行语义与通俗度审查**：
   * 阅读目标文件内容，根据上述三大标准评估现有注释的准确度与初学者易读性。
3. **输出审计报告**：
   在项目根目录下生成 `comments_report.md`。报告须包含以下部分：
   * **概要统计**：总行数、空白行、代码行、注释行、注释密度比例（含达标状态结论）。
   * **缺失清单**：列出缺少文档/注释的行号与函数名。
   * **内容一致性与通俗度审查**：针对具体代码片段指出哪些注释需要更正，哪些注释过于生硬需要改写成小白视角的通俗说明。
   * **修复示范**：提供一份修改后的代码注释片段示范。
4. **展示报告**：
   向用户呈现生成的报告核心内容及结论。
